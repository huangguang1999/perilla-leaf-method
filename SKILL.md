# 紫苏叶方法 · 证据 → 推断

> **紫苏叶理论**：走进顶级寿司店，所有人眼睛盯着的永远是金枪鱼大腹、海胆、鱼子酱——昂贵、显眼、性感。但真正让一整套料理成立的，可能是一片来自特定产地的小小紫苏叶：它不贵、不起眼、不在菜单最显眼处，却去腥、提香、定调。**没有金枪鱼，菜单少一道菜；没有紫苏叶，整套餐端不上来。**
>
> 判断任何事物也一样：**别只盯显眼又贵的「金枪鱼」，往下拆到那片「没它整套就端不上来」的紫苏叶**——决定性的、不可替代的底层环节。**从底层事实出发，不从叙事出发。** 全程一句话压舱：**「增长 / 这件事撞到墙的时候，墙在哪里？」**

## 一、内核（8 条）

1. **只对证据回应（从底层出发，不从叙事出发）** — 先剥掉情绪、最响的名字、最性感的叙事、第一印象，问「抛开这些，底层事实 / 硬限制到底说了什么」。
2. **证据→推断链** — 每步写成「因为[具体事实] → 所以[推断]」，串成结论。**永远给路径，不只给结论。**
3. **找那片紫苏叶（决定性因素）** — 挑出「没它整套就端不上来」的 1–3 个环节，**看谁最窄、不是看谁最大**。判定用 **紫苏叶三问**：① 它是不是（逻辑 / 物理上）非有不可？② 有没有可替代方案？③ 一旦压力 / 需求放大，它是不是最先崩、最难替补？三个都「是」→ 它就是决定项。
4. **剔除干扰项（重要 ≠ 决定性）** — 把「显眼、重要、但换掉也不翻盘」的金枪鱼们主动标出并排除，别让它们稀释判断。
5. **反驳对立面** — 把**最强**的相反结论也认真论一遍（甚至让 AI 来攻击自己的逻辑）：先假设自己可能错，看逻辑还站不站得住；钉不死的，留作必须承担的风险。挑对方最强的理由，不是最好驳的稻草人。
6. **随证据修正** — 证据变了就改判断；改主意、认错是对的，别为护住第一印象硬撑。
7. **不必等 100% 确定** — 用「倾向 + 把握（大 / 中 / 小）」表达，既诚实又够用。
8. **诚实标注来源** — 每条证据分清 ✅亲眼/查到 · ⚠️听说/待核实 · 💭我猜 · ❓不知道。不编；证据弱就把握小，语气不能替代证据。

## 二、操作流程（6 步）

1. **明确判断目标** — 一句话：判断什么、到什么程度（好不好 / 买不买 / 做不做 / 选 A 还是 B）。
2. **列证据** — 写出相关事实，每条标来源（✅/⚠️/💭/❓）；从底层、可核对的事实出发，别从叙事出发。
3. **逐条 证据→推断** — 每条「因为[事实] → 所以[推断]」，不跳步。
4. **标紫苏叶 + 剔金枪鱼** — 用紫苏叶三问圈出 1–3 个决定项（「撞墙点」）；把显眼但不决定的因素列出来、标「已排除」。
5. **反方一遍** — 写最强相反结论（可让 AI 攻击自己的逻辑），钉不死的标残留风险。
6. **汇成结论** — 倾向 + 把握（大/中/小）+ 一句话（那片紫苏叶 / 最决定性的理由）。

## 三、文字产出模板（直接套用）

```
【判断目标】一句话：判断什么、到什么程度。

【证据 → 推断】
1. 因为 [证据①] ✅/⚠️/💭/❓ → 所以 [推断①]
2. 因为 [证据②] …          → 所以 [推断②]
3. 因为 [证据③] …          → 所以 [推断③]

【紫苏叶 / 已剔除】
· 紫苏叶（决定项）：[环节] —— 若它反过来 → 结论变成 ___（三问：非有不可? 可替代? 压力放大谁先崩?）
· 金枪鱼（显眼但不决定，已排除）：___

【反方一遍】
· 最强相反结论：___（二选一时＝认真论证另一个更好）
· 用事实钉：因为 [证据] → 所以反方这点 站不住 / 站得住
· 钉不死的风险：___

【结论】倾向：好/不好（买/不买，做/不做，选 A/选 B）· 把握：大/中/小 · 一句话：___
```

## 四、思考路径图（HTML 成品图 —— 用户要「出图 / 思考路径 / 可视化 / 一张图看懂」时生成）

**直接复用 `templates/diagram-template.html` 的骨架、配色、类名**（可参考 `examples/`），按判断目标填充——别从零设计、别加新元素。

布局（纵向单脊柱，浅色笔记风）：
- **顶部**：判断目标（标题）。
- **主体**：从上到下一条脊柱；每步 = 上方「证据」卡 + 中间 ↓ + 下方「所以(推断)」卡；左侧 **纯数字 1 2 3（细体斜体衬线 Georgia、深色、略大、无边框、上下留白，用页面底色把脊柱线在数字处断开）**。
- **底部**：结论框（一句话；倾向用颜色点缀；把握写在 `结论 · 把握 X`）。
- **只允许**：判断目标 + 证据→推断步骤 + 结论。**别加**评分卡、把握色块、风险栏、图例等任何额外区块。证据来源标记（✅/⚠️/💭/❓）内联写在证据卡里。
- 紫苏叶（决定项）可在该步推断里用「（决定项）」轻标，不另开区块。

**输出**：写到 `~/perilla-leaf-diagrams/<目标简称>-<日期>.html`（避开 TCC 保护目录），生成后给用户路径、提示可 `open` 看 / 截图发飞书。

## 五、诚实边界

- **不编证据**：没有的事实标 ❓不知道，不用猜测冒充事实。
- **结论可被推翻**：新证据出现就按内核 6 改；改判断是方法生效，不是失败。
- **紫苏叶也有风险**：小、冷、窄的决定项往往伴随「信息不透明 / 一旦出现可替代路线就坍塌」的脆弱性——判断里要把这类风险写出来。
- **只给判断、不替人做主**：产出的是「基于现有证据的倾向 + 把握」，最终决定权在使用者。

## 六、常见误区

只盯金枪鱼、忘了找紫苏叶（被最响的名字 / 最性感的叙事带跑）· 堆一堆次要因素却不剔除 · 只论正方不反驳 · 把第一印象当结论。
